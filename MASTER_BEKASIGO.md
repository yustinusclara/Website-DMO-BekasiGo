Master Dokumen Final BekasiGo

1. Ringkasan Produk
   BekasiGo adalah website DMO publik Kota Bekasi yang dibangun secara modular, immersive, dan content-driven, dengan fitur:
   • website publik DMO
   • CMS internal
   • Smart Trip Planner berbasis Gemini API
   • Interactive Maps berbasis Google Maps API
   • Blog yang dikelola dari CMS
   • konten editorial city stories
   • integrasi aset visual/video Bekasi
   • workflow vibecoding penuh via AI Agent
   BekasiGo menggabungkan:
   • DNA utama: macaotourism.gov.mo
   • inspirasi DMO global: Tokyo, Seoul, Singapore, Taipei, Hong Kong, Busan
   • visual identity Bekasi: floating city, landmark 3D-isometric, heritage, urban lifestyle

---

2. Workflow Pengembangan
   Workflow yang dipakai:
1. Brainstorming di Emergent.sh
1. Build MVP di Emergent.sh
1. Push/deploy ke GitHub
1. Clone repo GitHub ke Antigravity IDE
1. Lanjut pengembangan dengan AI Agent di Antigravity
1. Gunakan Supabase untuk database + auth
1. Deploy frontend ke Vercel
1. Media delivery menggunakan Cloudinary

20. Prompt Master untuk AI Agent
    Prompt Homepage
    Bangun homepage website DMO Kota Bekasi bernama BekasiGo dengan DNA utama macaotourism.gov.mo. Gunakan nuansa East Asian DMO yang modern, immersive, youthful, cinematic, dan modular. Hero section wajib menggunakan video background full-screen dari Kota Bekasi Floating City. Setelah hero, tampilkan quick explore navigation, signature floating city visual, featured 3D landmarks Bekasi, heritage spotlight, urban lifestyle showcase, event highlights, practical travel utility, smart trip planner intro, interactive map preview, city stories, latest blog posts, dan footer mega navigation. Gunakan React + TypeScript dengan reusable components. Motion harus terinspirasi dari immersive follow, spotlight masking, dan GSAP pill navigation, tetapi tetap performant dan mobile-friendly.
    Prompt CMS
    Bangun admin CMS untuk BekasiGo menggunakan React + TypeScript dengan layout clean, modular, cepat, dan data-focused. Harus ada login, dashboard, homepage manager, destinations manager, events manager, restaurants manager, hotels manager, transport manager, stories manager, blog manager, media library, navigation manager, site settings, dan role-based access. Semua konten frontend harus editable dari CMS. Media harus mendukung Cloudinary URL input.
    Prompt Smart Trip Planner
    Bangun halaman Smart Trip Planner untuk BekasiGo yang terinspirasi dari tripplanner.macaotourism.gov.mo. User dapat memilih durasi perjalanan, minat, budget, starting point, dan travel style. Frontend memanggil FastAPI untuk mengambil data relevan dari Supabase dan membentuk context untuk Gemini API agar menghasilkan itinerary. Tampilkan hasil dalam bentuk timeline visual, destination cards, rekomendasi makanan, transport notes, dan Google Maps preview.
    Prompt Explore Map
    Bangun halaman Explore Map untuk BekasiGo menggunakan Google Maps API. Tampilkan marker untuk destinasi, restoran, hotel, dan transport. Sediakan filter kategori, panel list/grid pendamping, preview card saat marker diklik, dan CTA ke detail page. Desain harus bersih, mobile-friendly, dan menyatu dengan visual identity BekasiGo.
    Prompt Blog
    Bangun modul Blog untuk BekasiGo beserta CMS management-nya. Blog harus memiliki halaman listing, detail, kategori, dan tag. Di CMS harus ada Blog Manager dengan fitur create, edit, draft, publish, featured image, categories, tags, excerpt, rich text content, SEO metadata, related content linking, dan featured post toggle. Blog harus terhubung ke homepage melalui section Latest from Blog.

21. Aturan Keras untuk AI Agent
    • jangan bangun seperti SaaS template generik
    • jangan overuse animasi
    • jangan hardcode konten yang harus dikelola dari CMS
    • jangan biarkan Gemini menjawab tanpa grounding data internal
    • jangan membuat map tanpa list/grid pendamping
    • jangan membuat CMS terlalu dekoratif
    • jangan melupakan mobile responsiveness
    • jangan mencampur stories dan blog sebagai satu modul yang kacau
